# /start — начало рабочей сессии

## Шаг 1. Загрузить контекст (обязательно, ничего не пропускать)

Прочитай в этом порядке:
1. `CLAUDE.md` — правила проекта
2. `docs/TZ.md` — ТЗ: схема БД, все ~137 endpoint'ов, безопасность, cron
3. `docs/ROADMAP.md` — план: 14 фаз
4. `.claude/sessions/` — **последний по дате** лог сессии (если есть). Если папки нет — это первая сессия.
5. `docs/BACKEND_BUGS.md` (если есть) — что уже найдено

Затем: `git status` + `git log --oneline -5` — понять, в каком состоянии код.

## Шаг 2. Определить следующую фазу

- Найди в `docs/ROADMAP.md` **первую фазу, где есть невыполненные `[ ]`**
- Если предыдущая сессия оставила блокеры — сначала разбери их
- Если фаза уже частично сделана — продолжи с первого `[ ]`

## Шаг 3. Показать план и получить подтверждение

Выведи коротко:
```
Последняя сессия: <дата> — <что сделано>
Текущее состояние: git <ветка>, <N> файлов в working tree
Следующая: ФАЗА <N> — <название>
Задачи: <список пунктов [ ] из ROADMAP>
Endpoints: <сколько закрывается>
Блокеры с прошлой сессии: <если есть>
```
Спроси: **«Начинаю Фазу N?»** — и жди ответа.

## Шаг 4. Выполнить фазу

Правила выполнения:
- **ТАХМИН МАНЪ.** Каждый endpoint проверяется **реальным запросом** (curl / Postman / Supertest), а не «должен работать».
- Одна сессия = **одна фаза**. Не забегай вперёд.
- Модуль = controller + service + dto. Бизнес-логика **только** в service.
- Формат ответа всегда `{ data, errors, statusCode }` (ResponseInterceptor).
- `errors` заполняется **только при ошибке** — никогда `["success"]`.
- Пагинация — курсорная. Guards — Jwt / Owner / Roles / Block / Privacy.
- TypeScript strict, `any` запрещён.
- Каждый новый endpoint сразу попадает в Swagger (`@ApiOperation`, `@ApiResponse`, DTO).

## Шаг 5. Проверка перед завершением

Обязательно:
```bash
npm run build
npm run lint
npx prisma validate
npm run test:e2e   # если тесты для этой фазы есть
```
Плюс **живая проверка**: подними `docker compose up -d`, дёрни новые endpoint'ы curl'ом, покажи реальные ответы.

## Шаг 6. Отчёт

В конце дай отчёт:
- Что построено (список endpoint'ов + модулей)
- Что **реально проверено** живыми запросами (с ответами)
- Что отличается от ТЗ и почему
- Новые баги / решения
- Что осталось незакрытым
- Отметь `[x]` в `docs/ROADMAP.md`

Затем напомни: **«Сессию можно закрыть командой /stop»**.
