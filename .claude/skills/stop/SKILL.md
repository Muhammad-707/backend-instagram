# /stop — завершение рабочей сессии

Выполняй строго по шагам. **Ничего не пропускать.**

## Шаг 1. Проверка качества (перед коммитом!)

```bash
npm run build
npm run lint
npx prisma validate
```
**Если что-то красное — сначала чиним, потом коммитим.** Не коммить сломанный билд.
Если починить в этой сессии нельзя — запиши в блокеры и **спроси пользователя**, коммитить ли как есть.

## Шаг 2. Обновить документы

1. **`docs/ROADMAP.md`**
   - Отметить `[x]` всё, что реально сделано и проверено живым запросом
   - Под фазой добавить блок `> Заметки Фазы N:` — важные решения, отличия от ТЗ, ограничения
   - Незакрытые пункты оставить `[ ]` (не врать себе)

2. **`docs/BACKEND_BUGS.md`** (создать, если нет) — баги, найденные в **нашем** коде и в сторонних сервисах

3. **`docs/API_SPEC.md`** (если менялись контракты) — актуальные DTO

## Шаг 3. Записать лог сессии

Создай `.claude/sessions/YYYY-MM-DD-HHmm.md`:

```md
# Сессия YYYY-MM-DD

## Фаза
ФАЗА N — <название>  ·  статус: завершена / частично

## Сделано
- <модуль/endpoint> — что именно
- ...

## Проверено живыми запросами
- POST /api/... → 201, ответ: {...}
- ...

## Решения (почему так, а не иначе)
- ...

## Отличия от ТЗ
- ...

## Блокеры / незакрытое
- ...

## Следующий шаг
ФАЗА N+1 — <название>. Начать с: <первый пункт [ ]>
```

## Шаг 4. Git: add → commit → push

```bash
git status
git add .
git commit -m "Phase N: <краткое описание> (<M> endpoints)"
git push
```

Правила коммита:
- Сообщение — по-английски, в формате: `Phase 6: posts, comments, reels (20 endpoints)`
- Если это первая сессия и remote не настроен — **скажи об этом пользователю** и подскажи:
  ```bash
  git remote add origin https://github.com/<username>/instagram-backend.git
  git branch -M main
  git push -u origin main
  ```
- Если `git push` упал (нет прав / нет remote / конфликт) — **не молчи**: покажи ошибку и предложи решение.
- `.env`, `node_modules/`, `dist/`, `uploads/` — должны быть в `.gitignore`. Проверь перед `git add .`
  Если `.env` попал в индекс — **немедленно убрать** (`git rm --cached .env`) и предупредить.
