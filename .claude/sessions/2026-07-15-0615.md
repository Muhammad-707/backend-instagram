## Проверено живыми запросами

### Стриминг (главное требование)
| Запрос | Ответ |
|---|---|
| без Range | `200`, весь файл **8 945 229 байт** |
| `bytes=0-1023` | **`206`**, `Content-Range: bytes 0-1023/8945229` |
| `bytes=5000000-` (перемотка) | `206`, 3 945 229 байт |
| `bytes=-500` (хвост) | `206`, 500 байт |

- **Файл реально играет:** скачанный по `streamUrl` mp3 → ffprobe: `mp3, 44100 Hz, 2 канала, 192 kbps, 373с` —
  длительность совпадает с БД.
- **Байты диапазонов идентичны оригиналу** (сравнение буферов с исходным файлом → `true`).
- `/music/:id/stream` **публичный**: `206` без токена. Остальные endpoint'ы без токена → `401`.

## Решения (почему так, а не иначе)
- **`streamUrl` вместо прямой ссылки в S3.** Клиенту отдаём свой endpoint: Range проходит через наш код,
  и позже можно будет считать прослушивания.
- **`/stream` помечен `@Public()`** — тег `<audio src="...">` в браузере не умеет слать `Authorization`.
  Ничего нового не открываем: объекты в MinIO и так публичны на чтение.
- **Неразбираемый `Range` → `200` целиком**, а не `416`: так предписывает HTTP-спецификация.
- **`getPartialObject` вместо чтения всего файла**: иначе перемотка трека тянула бы все 9 МБ с начала.
- **Заглушки из seed удалены, а не оставлены** — поиск не должен отдавать треки, которые не играют.
- **upsert по паре title+artist**: `@@unique` в схеме нет, поэтому ищем вручную — повторный `music:import` не плодит дубли.
- **Скрипт импорта вместо хардкода треков** — когда появятся 34 mp3 с Pixabay, код менять не придётся.
