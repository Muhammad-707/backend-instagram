## Решения (почему так, а не иначе)
- **BullMQ основной, cron подстраховка.** Задача с delay=24ч, jobId=story-N (от дублей). Процессор перед
  удалением проверяет _count.highlights. Cron раз в час чистит истёкшие не-актуальные — если Redis был недоступен.
- **Мультизагрузка = N отдельных Story** (не одна с N медиа): в IG каждая история своя, со своим таймером и зрителями.
- **Реакция/ответ = Message в чате** (по схеме, messageId в StoryReaction/StoryReply).
- **isViewed/allViewed только на сервере** через StoryView (@@unique storyId+userId).
- **close-friends истории** видны только тем, кого автор добавил в близкие (проверка в loadVisible и rail).
- **Мерил через 127.0.0.1** (localhost в Windows добавляет ~200мс — известно с Фазы 6).
