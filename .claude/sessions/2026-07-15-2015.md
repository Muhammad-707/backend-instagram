## Проверено живыми запросами (4 сокет-клиента: host+follower+viewer+guest, LiveKit-токены декодированы)
- **start** → 201, host publisher-токен canPublish=**true**, wsUrl=ws://localhost:7880.
- **follower** получил `live:started` в сокет; эфир в `GET /live/feed`.
- **viewer join** → subscriber-токен canPublish=**false**, viewersCount=1.
- **comment**=1, **like×3**→likesCount=3 (3 события `live:like`), **reaction**=1 — всё в комнату.
- **request-join** → хост получил `live:join-request`.
- **accept** → гость получил `live:join-accepted` с токеном canPublish=**true** (split-экран) + `live:guest-joined`.
- **decline** → гость получил `live:join-declined`.
- **camera off** → isCameraOn=false (сокет `live:camera`); **audio** → isAudioOn=true (звук идёт всегда).
- **viewers** → список; **kick** → зритель получил `live:kicked`.
- **Block:** заблокированный komron `join` → **403**.
- **stats** (likes=3, comments=1, reactions=1, peak=1, duration); **end** → `live:ended`, статус в БД = ENDED.
- Все 12 групп проверок — PASS. Swagger: 18 /live routes.

## Решения (почему так, а не иначе)
- **Отдельный namespace /live** (не /rt) + свой мост LiveRealtimeService: комнаты эфиров и личные live-события изолированы от чата/присутствия.
- **Токен гостю приватно** (в liveuser:<id>, не в комнату) — split-экран, чужие токен не видят.
- **Камера ≠ звук:** PUT /camera меняет только видео; isAudioOn отдельным PUT /audio — «звук идёт всегда».
- **LiveKit best-effort** (closeRoom/removeParticipant в try/catch) — сбой SFU не роняет REST.
- Токены подписаны локально (JWT_SECRET) для теста; socket.io-client через --no-save.
