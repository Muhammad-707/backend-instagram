## Проверено живыми запросами
- `check-username` → `{available:true}` для свободного, `{available:false}` для занятого (`eraj` из seed)
- `register` → `201` + пара токенов + профиль. `passwordHash` наружу **не утекает**
- `register` негатив: пароли не совпадают → `400` · возраст < 13 → `400` · userName занят → `409`
- `login` по **userName**, **email**, **phone** → все три `200`
- `login` с неверным паролем → **`401`, а не 500** (баг старого API)
- **Rate-limit: 5 логинов проходят, 6-й и 7-й → `429`** · `resend-code` второй раз подряд → `429`
- `GET /auth/me` с токеном → `200` + профиль; без токена → `401`
- **`POST /upload` без токена → `401`**; с токеном → `201`
- `refresh` → новая пара, старый отозван; **повторное использование старого → `401` + ВСЕ сессии сброшены**
- `logout` → `200`, затем `refresh` → `401`; повторный `logout` → `200` (идемпотентно)
- **`forgot-password` → письмо РЕАЛЬНО ушло в MailHog**: To `eraj.dev@example.com`, From `no-reply@instagram.local`,
  Subject «433684 — код для сброса пароля Instagram». HTML: логотип + градиент + **6 плиток с цифрами `433684`** + «15 минут»
- Несуществующий email → **тот же ответ**, письма нет (нет user enumeration)
- `verify-code`: неверный код → `400` · **просроченный (сдвинул `expiresAt` в БД на −20 мин) → `400`** · повторный → `400`
- `reset-password` → `200`; **старый пароль → `401`, новый → `200`**; повторный resetToken → `400` (одноразовый)
- `change-password`: неверный старый → `401`; верный → `200` + все refresh-сессии отозваны
- Swagger `/api/docs-json` — **14 endpoint'ов**, замок 🔒 ровно на защищённых
- `npm run build` · `npm run lint` · `npx prisma validate` → **зелёные**

## Решения (почему так, а не иначе)
- **JwtAuthGuard глобальный (`APP_GUARD`), всё закрыто по умолчанию**, открывается точечно `@Public()`.
  При 137 endpoint'ах «забыть повесить guard» — вопрос времени; глобальный запрет делает дыру невозможной.
  Именно это закрыло `/upload` без правок в его контроллере.
- **Refresh-токен хранится как SHA-256**, не в открытом виде: утечка таблицы `RefreshToken` не даст войти в аккаунты.
- **Детект кражи токена**: повторное использование уже отозванного refresh → гасим ВСЕ сессии юзера.
- **resetToken одноразовый через Redis**: JWT сам по себе переиспользуем, поэтому `jti` кладётся в Redis (TTL 15 мин)
  и удаляется при первом `reset-password`. Плюс claim `typ: 'reset'` — обычным access-токеном пароль не сменить.
- **Нет user enumeration**: `forgot-password` отвечает одинаково для существующего и несуществующего email;
  `verify-code` даёт один текст на «код неверен», «код чужой», «код просрочен».
- **Смена пароля (и reset, и change) отзывает все refresh-сессии** — украденный токен перестаёт работать.
- **Профиль создаётся сразу при регистрации** (`profile: { create: {} }`) — иначе `/auth/me` и код Фазы 4
  спотыкались бы о `profile: null`.
- **`auth` в nodemailer не отправляется при пустом `SMTP_USER`** — иначе MailHog рвёт соединение (баг #9).

## Отличия от ТЗ
- `register` возвращает **сразу пару токенов** (в ТЗ не сказано) — иначе фронту нужен второй запрос на логин.
- `register` проверяет **возраст 13+** (в ТЗ нет, но `dob` там есть, а IG это требует).
- Регистрация **только по телефону невозможна** — email обязателен (решение пользователя, см. выше).
- `EmailCodeType.VERIFY_EMAIL` в схеме есть, но подтверждение email не реализовано — его нет в списке §5.1.
