# ROADMAP — Frontend (Фаза 14: подключение к новому backend'у)

> ⚠️ **Этот документ — для РЕПОЗИТОРИЯ FRONTEND, не для backend'а.**
> Он описывает миграцию существующего фронта (Next.js) со старого **softclub-API** на наш
> новый backend (167 endpoints, конверт `{ data, errors, statusCode }`, cursor-пагинация,
> Socket.IO, LiveKit).
>
> 🔒 **Начинать эту фазу можно ТОЛЬКО после ✅ Фазы 13** (backend полностью готов и проверен:
> e2e 16/16, Swagger `/api/docs`, `docs/swagger.json` — точный источник типов).
> До этого фронт трогать не нужно — контракт API ещё может уточняться.

**Точный контракт API** — всегда из Swagger нового backend'а: `http://localhost:3000/api/docs`
или файл `docs/swagger.json` в backend-репозитории. Не выдумывать поля — сверяться со Swagger.

Правило то же, что на backend'е: **не тахмин, а проверка живым экраном.** Каждая подфаза
заканчивается кликом по реальному UI, а не «должно работать».

- [ ] **Карта миграции** `docs/MIGRATION_MAP.md`: таблица «старый endpoint softclub → новый endpoint + изменения полей». Основа — 21 баг из backend `docs/TZ.md §2` (что именно поменялось в контракте)
- [ ] Инвентаризация: список всех мест, где фронт дёргает API (grep по axios/fetch), — чтобы ничего не пропустить

> Ориентир: подфаза не пишет фич, только готовит почву и фиксирует точку отката.

## Фаза 14.1 — Слой API (axios, конверт, cursor, типы, refresh)
- [ ] `lib/axios.ts`: `baseURL` из env; **response-интерцептор разворачивает конверт** — `res.data.data` наружу, `res.data.errors` → `throw` с текстом (единая обработка ошибок)
- [ ] Заголовок `Authorization: Bearer <accessToken>` из хранилища токенов
- [ ] **Автоматический refresh:** на `401` — один раз дёрнуть `POST /auth/refresh` (ротация!), сохранить новую пару, повторить исходный запрос; при повторном `401` — разлогин. Учесть гонки (очередь ожидающих запросов на время refresh)
- [ ] **Cursor-пагинация вместо `PageNumber/PageSize`:** хелпер `fetchPage({ cursor, limit })` → `{ items, nextCursor, hasMore }`; заменить пагинаторы на «Load more»/infinite-scroll по `nextCursor`
- [ ] **Типы из Swagger:** сгенерировать TS-типы из `swagger.json` (напр. `openapi-typescript`) → `types/api.ts`; сервисы переводятся на эти типы, старые DTO удаляются
- [ ] Проверка: логин → токен сохранён → защищённый запрос → 200; протухший access → тихий refresh → запрос прошёл; лента листается курсором (стр.1 ≠ стр.2)

> Это фундамент — все следующие подфазы опираются на этот слой. Баги softclub #1 (конверт),
> #3/#4/#21 (пагинация), #7 (голый массив) закрываются здесь на клиенте.

## Фаза 14.2 — Socket.IO вместо polling
- [ ] `lib/socket.ts`: подключение к namespace `/rt`, авторизация access-JWT в `handshake.auth.token`, реконнект, обновление токена
- [ ] **Чат realtime:** подписка на `message:new|edited|deleted`, `message:reaction`, `message:read`; убрать интервальный polling переписки
- [ ] **Presence:** `presence:update` → индикатор «в сети / был(а) N мин назад» (`isOnline`, `lastSeenAt`); heartbeat со стороны клиента
- [ ] **Typing:** отправка `typing:start/stop`, отрисовка «печатает…» у собеседника
- [ ] **Уведомления в реальном времени:** приём `notification:new` = `{ notification, unreadCount }` → бейдж + тост (экран уведомлений — в 14.3)
- [ ] Проверка **с двух клиентов**: сообщение долетает мгновенно (не polling), typing/online/reaction/read видны у второго; реконнект после разрыва

> Баг softclub #14 (нет realtime) закрывается. Socket.IO — источник живых событий,
> REST остаётся для истории/первичной загрузки.

## Фаза 14.3 — Новые экраны: Уведомления · Актуальное · Заметки
- [ ] **Уведомления** (`GET /notifications`, cursor): лента 17 типов с группировкой («user1 и ещё 5…»), `unread-count`, `read`, `read-all`, экран «кто заходил в профиль» (`profile-views`). Живой пуш из 14.2
- [ ] **Актуальное / Highlights** (`/highlights` CRUD): обложки на профиле, создание из историй, просмотр, редактирование состава/названия, удаление
- [ ] **Заметки / Notes v2** (`/notes`): пузырёк заметки над аватаром в чат-листе и профиле; создание (текст ≤60, музыка, цвет фона, TTL 24ч), лайк заметки, ответ → уходит в директ (превью-снимок текста)
- [ ] Проверка: уведомление о лайке приходит живьём и появляется в списке; актуальное переживает 24ч; ответ на заметку открывает чат с превью

> Экранов этих на старом фронте не было вовсе — API под них готов (backend Фазы 7, 8, 10).

## Фаза 14.4 — Истории v2 + Приватность
- [ ] **Истории v2** (`/stories`): мультизагрузка до 10 файлов за раз → N историй; фото **и видео**; музыка + `musicStartSec`; оверлеи (текст/эффекты, `overlays` JSON); фильтры; `closeFriendsOnly` (зелёное кольцо)
- [ ] Рейл историй с `isViewed`/`allViewed` **с сервера** (баг #17), лайк-boolean (баг #15), реакции и ответы (уходят в чат), **список зрителей** автору (баг #16)
- [ ] Репост поста в историю (`fromPostId`)
- [ ] **Приватность:** закрытый аккаунт (toggle), экран заявок на подписку (accept/decline), близкие друзья (add/remove/list), блокировки (block/unblock/list). Скрытие контента приватных аккаунтов на клиенте согласно `canViewContent`
- [ ] Проверка: история с музыкой и текстом создаётся и играет; close-friends история видна только близким; заявка на приватный → после accept контент открывается; заблокированный исчезает из поиска/ленты

> Баги softclub #15, #16, #17 закрываются на UI. Приватность/блок/близкие друзья — новые экраны.

## Фаза 14.5 — Чат v2
- [ ] **Реакции** на сообщения (`POST/DELETE .../reaction`) + живое `message:reaction`
- [ ] **Редактирование** (`PUT /chats/messages/:id`, ≤15 мин) с меткой `editedAt`; **удаление только своего** (баг #11) — чужое не удалить
- [ ] **Голосовые** (запись → отправка `audio`-файлом) + плеер с длительностью
- [ ] **bulk-delete** (выделение нескольких своих сообщений), ответы (`replyToId`), отправка поста в чат (`sharedPostId`), стикеры
- [ ] **Запросы на переписку:** входящие от неподписанных в «Запросы» (accept/decline); список чатов с `lastMessage`, `lastMessageAt`, `unreadCount`, `peer`, `isOnline` (баг #13)
- [ ] «Просмотрено» (`POST /chats/:id/read`), темы, никнеймы, mute
- [ ] Проверка: реакция/редактирование/удаление видны у собеседника мгновенно; чужое сообщение удалить нельзя; голосовое записывается и играет; заявка на переписку не плодит дубли после decline

> Баги softclub #11 (удаление чужого), #12 (опечатки полей), #13 (пустой чат-лист) закрываются.

## Фаза 14.6 — LIVE (прямые эфиры)
- [ ] `livekit-client` в проект; подключение к комнате по токену из `POST /live/start` (хост, publisher) и `POST /live/:id/join` (зритель, subscriber)
- [ ] Namespace `/live`: подписка на `live:started` (рейл), `live:viewers`, `live:comment`, `live:like`, `live:reaction`, `live:ended`
- [ ] **UI хоста:** старт/стоп эфира, переключение камеры (видео выкл → обложка, **звук идёт всегда**), микрофон, счётчики зрителей/лайков, кик зрителя, статистика по завершении
- [ ] **UI зрителя:** просмотр видео, комментарии в эфир, лайки (всплывающие сердечки, можно много раз), реакции-смайлы
- [ ] **Гости и split-экран:** заявка `request-join` → хост accept → гость становится вторым publisher (`live:guest-joined`), split-раскладка; decline → уведомление отказа
- [ ] Доступ: подписчик видит эфир в рейле; не подписчик — через поиск/профиль («В эфире»); закрытый хост — только подписчикам; заблокированный — 403
- [ ] Проверка **в браузере** (реальный WebRTC): хост стримит, зритель видит и комментирует, гость выходит в split-экран, камера off → обложка + звук идёт

> Полностью новая фича (backend Фаза 12.5). Требует настоящей проверки в браузере с камерой/микрофоном.

## Фаза 14.7 — Верификация, музыка, финальная полировка
- [ ] **Верификация** (`/verification`): экран статуса, «7 дней бесплатно» (1 раз), mock-подписка ($1000/мес), отмена; галочка на профиле; уведомление за день до конца триала
- [ ] **Музыка в постах и историях** (`/music`): выбор трека при создании, поиск/тренды, стриминг с Range в плеере, сохранённая музыка в профиле, `usesCount`
- [ ] **Звонки** (WebRTC-сигналинг через сокет `call:offer/answer/ice/end`) — если в объёме релиза
- [ ] Финальная полировка: скелетоны/лоадеры на cursor-загрузках, единый показ ошибок из `errors[]`, пустые состояния, доступность, ретраи сокета
- [ ] Регрессия: пройти все экраны старого фронта — ничего не сломалось при переезде на новый API
- [ ] Слить `backend-migration` → `main`, тег `v2.0-fullstack`
